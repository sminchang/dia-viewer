# dia-viewer

A pure **diagram manifest renderer**. Loads a manifest JSON (ERD or C4 architecture), renders it interactively with ReactFlow, lets you drag nodes and switch layouts, and exports PNG/SVG.

No database connection, no credentials, no save-to-server — it only reads a manifest and produces an image. Safe to run locally or host as static files.

## Run

```bash
npm install
npm run dev          # http://localhost:5173
```

Then: **예시 ERD** / **예시 아키텍처** buttons load the bundled examples, or **매니페스트 열기** loads your own manifest JSON.

## What it does

| Feature | |
|---------|---|
| ERD rendering | tables with per-column PK/FK/UK badges, column-level FK edges, crow's-foot via cardinality labels |
| Architecture rendering | C4 person/system/container/component nodes, boundary groups, labelled relationships |
| Layout | 계층 (ELK hierarchical) / 중앙 (d3-force radial) toggle — works for both kinds |
| Edit | drag nodes; ERD edge handles re-pick sides on drop |
| Export | PNG / SVG at full diagram bounds |

## Input contract

Consumes the diagram manifest contract — `manifest.schema.json` (validation) and `src/manifest.ts` (types), documented in `SPEC.md`. Generated by the `create-erd` and `create-diagram` skills. One manifest = one diagram.

## Build

```bash
npm run typecheck
npm run build        # → dist/ (static, deployable anywhere)
```

## Structure

```
manifest.schema.json       # contract — JSON Schema (skills target it)
SPEC.md                    # contract — design rationale + field reference
src/
├── manifest.ts            # contract — TypeScript types (canonical for the app)
├── core/
│   ├── manifestToFlow.ts  # manifest → ReactFlow nodes/edges + column handles
│   ├── layout.ts          # ELK (hierarchical) + d3-force (central)
│   └── exportImage.ts     # PNG/SVG via html-to-image
├── nodes/
│   ├── TableNode.tsx      # ERD table
│   ├── C4Node.tsx         # C4 element
│   └── GroupNode.tsx      # domain/boundary background
└── App.tsx                # load · layout · drag · export
```
